Daily Update Process (5 Steps)

1. Make Your Changes

Edit files in: C:\Documents\GitHub\DeepFish\

2. Open GitHub Desktop

Your changes will automatically appear in the "Changes" tab

3. Commit Your Changes

  • Review the changed files (left panel)
  • Add a commit message (bottom left text box)
    • Example: "Fixed Hanna's prompt" or "Updated landing page"
  • Click "Commit to main" (blue button)

4. Push to GitHub

  • Top button changes from "Fetch origin" to "Push origin"
  • Click "Push origin"
  • Your changes are now on GitHub!

5. Railway Auto-Deploys

  • Railway detects the push automatically
  • Check Railway dashboard to watch deployment
  • Wait 2-5 minutes for build to complete
  • Your changes are live at deepfish.app!

üîß Common Scenarios

Scenario: I made a mistake

  • In GitHub Desktop: Repository ‚Üí Revert commit
  • Or: Roll back to any previous commit in History tab

Scenario: Want to see what changed

  • GitHub Desktop shows all diffs (green = added, red = removed)

Scenario: Railway deployment failed

  • Check Railway logs for error messages
  • Fix the error locally
  • Commit + push again

Scenario: Need to work on multiple things

  • Git handles it! Just commit when each part is done
  • You can commit multiple times before pushing
