üé• Oracle Video Analysis Workflow

Use Case: Watching YouTube videos (or any video content) and using insights to modify agent behavior or add new capabilities.

Step 1: Provide Oracle with Video URL

In Oracle mode, send the YouTube URL or video link:

Oracle, please analyze this video and extract key insights:
https://www.youtube.com/watch?v=example

Focus on: [specific topics or techniques you want Oracle to learn]
üí° Tip: Be specific about what you want Oracle to focus on. Examples:
  • "Extract the coding patterns shown"
  • "Identify the UI design principles discussed"
  • "Note the workflow automation techniques"

Step 2: Oracle Processes & Distills

Oracle will:

  • Access and analyze the video content
  • Extract relevant technical concepts
  • Identify actionable insights
  • Format findings in terminal-style output
ANALYSIS COMPLETE
-------------------
KEY CONCEPTS EXTRACTED: 7
ACTIONABLE ITEMS: 4
AGENT MODIFICATION RECOMMENDATIONS: 2

Step 3: Apply Insights to Agents

Use Oracle to modify agent .fsh files or create new training data:

Oracle: Apply these insights to Hanna's design process
- Update basePrompt with new UI principles
- Add capability: [specific new skill]
- Modify workflow to include: [new step]
‚ö†Ô∏è Important: Oracle modifications create new training data in agent.user.json files. Factory .fsh files remain unchanged unless you explicitly request system-level updates.

Step 4: Test & Validate

After Oracle makes changes:

  1. Review the updated .user.json file
  2. Test the agent with a sample task
  3. Verify the new behavior is working as expected
  4. Iterate with Oracle if adjustments needed

Step 5: Commit Changes

Once satisfied with modifications:

git add agents/[agent-name].user.json
git commit -m "Enhanced [agent] with insights from [video topic]"
git push

Railway will automatically deploy the updated agent training.

üìö Agent Training Best Practices

1. Ephemeral Training Materials

Remember: Videos, PDFs, and URLs are processed and discarded. Only distilled knowledge is saved to .user.json

2. Iterative Refinement

Train agents gradually. Add one capability at a time, test, then add more.

3. Factory Reset Available

Delete agent.user.json to return agent to factory settings. Factory .fsh files are always preserved.

4. Version Control

Git tracks all agent training evolution. You can roll back to any previous state.
